Widget component using dynamic projected content.

Projected title from parent component

The widget Title is using the ngProjectAs="[slot]='header' attribute so it's child component can select it using <ng-content select="[slot]='header'"></ng-content>.

The use of ngProjectAs it's optional and slot="header" could be used instead, but ngProjectAs makes multi layered projection possible, so the content can be re projected to a grandchild component.

Conent reprojection can be observed on the widget footer, which is projecting a button from the container component into the lib-widget component and then once more from lib-widget into the lib-widget-footer component.

The widget body does not use any attribute and any content will be projected inside of it by default.
